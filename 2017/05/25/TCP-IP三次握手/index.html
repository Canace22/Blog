TCP/IP三次握手

May 25, 2017 | web | 阅读

  1. 第一次 握手：客户端向服务器端发包，并且不确定服务器端是否收到，同时服务器端确认接受包；

  2. 第二次握手：服务器端向客户端发包作为回复，并且不确定客户端是否能接收自己发出的包，同时客户端收到包；

  3. 客户端发包回应服务器端，服务器端确认收到包，两边都确认能接收传递信息，通过快速连接，客户端发出初始信息，开始通信;
    用简洁易懂的话说就是：

    A：在吗？亲（第一次握手）

B：恩，有啥事？（第二次握手）

A：我们来聊天吧。（第三次握手）
